求解斐波那契数列的第n项

问题描述

先前在知乎看到一个面试题，分享一下

1、递推方程 f(n) = f(n - 1) + f(n - 2) ，已知 f(1) = 1，f(2) = 1，求第n项 （n <= 30）

拿到题一看，简单啊，这不就是前几篇说的，记忆化搜索，或者递推解决

2、递推方程 f(n) = f(n - 1) + f(n - 2) ，已知 f(1) = 1，f(2) = 1，求第n项 （n <= 80）

n最大能到80，第80项斐波那契数是什么？是23416728348467685。这么长的数字，妥妥的超过了int范围了。变量范围溢出，是典型的低级错误。面试过程中喜欢考这种细节。

3、递推方程 f(n) = f(n - 1) + f(n - 2) ，已知 f(1) = 1，f(2) = 1，求第n项 （n <= 10000），结果对10^9 + 7取模

这个问题没什么坑，n非常大，但结果对10^9 + 7求模，10^9 + 7是int范围内，因此不需要使用64位长整型。把问题1的代码拿来改改，加上取模即可

4、递推方程 f(n) = f(n - 1) + f(n - 2) ，已知 f(1) = 1，f(2) = 1，求第n项 （n <=1018），结果对109 + 7取模

注意了，n特别大，如果按照一般的递推方式，从1递推到n，那是不可取的
斐波那契数列是有通项公式的，但是通项公式底是个无理数，再加上f(n)数字特别大，精度无法满足
这个就是下面要分享的做法

快速幂算法

问题：求解的值，b <= 10 ^ 18

循环b次，不断累乘a的做法肯定是行不通的，因为b特别大

我们推导一下，

当b为偶数时，令b = 2c
那么，我们只需要求，就可以以o(1)的时间复杂度得到

当b为奇数时，令b = 2c + 1
那么，我们同样只需要求，就可以以o(1)的时间复杂度得到

不断的求解子问题，从而得到的结果。由于子问题的c是问题a^ b的b的一半，因此这个算法复杂度是o(log b)
